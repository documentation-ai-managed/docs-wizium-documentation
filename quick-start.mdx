## Setting Up Your Account

You create your WIZIUM account quickly and securely. The registration process takes just a few steps and gives you immediate access to the platform.

<Steps>
  <Step title="Visit WIZIUM" icon="globe" title-type="p">
    Navigate to the WIZIUM website and click the "Sign Up" button. You choose between email registration or SSO with your organization's provider.
  </Step>

  <Step title="Enter Details" icon="user" title-type="p">
    Provide your email address and create a secure password. If using SSO, you authenticate through your company's identity system.
  </Step>

  <Step title="Verify Email" icon="mail" title-type="p">
    Check your email for a verification link. You click the link to activate your account and complete the setup process.
  </Step>

  <Step title="Customize Profile" icon="settings" title-type="p">
    Add your name, profile picture, and team information. You set up notification preferences and security settings.
  </Step>
</Steps>

<Callout kind="info" collapsed="false">
  Use a strong, unique password and enable two-factor authentication for enhanced security.
</Callout>

## Creating Your First Project

You establish your initial documentation project to begin organizing content. The project creation wizard guides you through the essential setup steps.

<Tabs>
  <Tab title="Web Interface" icon="monitor">
    Use the web dashboard to create projects visually. You access intuitive forms and drag-and-drop functionality for a smooth experience.
  </Tab>

  <Tab title="API" icon="code">
    Automate project creation with the WIZIUM API. You write scripts to set up multiple projects programmatically.
  </Tab>
</Tabs>

<CodeGroup show-lines="true" tabs={[]}>
  ```bash
  curl -X POST https://api.wizium.com/projects \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "My First Project",
    "description": "Documentation for my product",
    "visibility": "private"
  }'
  ```

  ```javascript
  const wizium = require('wizium-sdk');

  async function createProject() {
  const project = await wizium.projects.create({
    name: 'My First Project',
    description: 'Documentation for my product',
    visibility: 'private'
  });

  console.log('Project created:', project.id);
  }

  createProject();
  ```
</CodeGroup>

## Importing Existing Content

You migrate your current documentation to WIZIUM seamlessly. The platform supports multiple import methods to preserve your existing content structure.

<Columns cols="2">
  <Card title="Git Repository" href="#git-import" icon="git-branch" horizontal="false">
    Connect your Git repository to import Markdown files automatically. WIZIUM maintains file structure and version history.
  </Card>

  <Card title="File Upload" href="#file-upload" icon="upload" horizontal="false">
    Upload individual files or ZIP archives. You organize content during the import process.
  </Card>
</Columns>

<Expandable title="Git Repository Import Details" default-open="false">
  You connect your Git repository by providing the repository URL and authentication credentials. WIZIUM pulls all Markdown files and preserves the directory structure.

  ```bash
  git clone https://github.com/your-org/docs.git
  cd docs
  wizium import --project-id YOUR_PROJECT_ID --source .
  ```
</Expandable>

<Expandable title="File Upload Process" default-open="false">
  You select files from your local system and upload them to WIZIUM. The platform automatically detects file types and converts content to the appropriate format.
</Expandable>

<Callout kind="tip" collapsed="false">
  Start with a small subset of documentation to test the import process before migrating your entire knowledge base.
</Callout>

<Steps>

  </Step>

  <Step title="Browse Projects" icon="folder" title-type="p">

  </Step>

  <Step title="Access Settings" icon="settings" title-type="p">

  </Step>
</Steps>
